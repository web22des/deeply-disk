# Компонент FeatureList

Компонент для отображения списка характеристик или особенностей с опциональными иконками. Идеально подходит для карточек товаров, курсов, услуг или любого контента, где нужно показать перечень пунктов с визуальным выделением.

## 📦 Установка

Компонент уже находится в проекте по пути: `src/components/ui/FeatureList/FeatureList.astro`

## 🎯 Базовое использование

```astro
---
import FeatureList from '@components/ui/FeatureList/FeatureList.astro';
---

<FeatureList
  items={[
    { text: '10 часов видео' },
    { text: '5 практических заданий' },
    { text: 'Сертификат по окончании' }
  ]}
/>
```

## ✨ Возможности

- ✔️ Отображение списка характеристик
- ✔️ Поддержка иконок для каждого пункта (из иконочного шрифта)
- ✔️ Адаптивная верстка (использует flex)
- ✔️ Поддержка кастомизации через CSS классы
- ✔️ Доступность (ARIA-атрибуты)

## 📋 Props (свойства)

| Свойство    | Тип             | Обязательное | По умолчанию     | Описание                       |
| ----------- | --------------- | ------------ | ---------------- | ------------------------------ |
| `items`     | `FeatureItem[]` | ✅           | -                | Массив пунктов для отображения |
| `class`     | `string`        | ❌           | -                | Дополнительные CSS классы      |
| `ariaLabel` | `string`        | ❌           | "Характеристики" | Метка для скринридеров         |

## Тип FeatureItem

```ts
interface FeatureItem {
  text: string; // Текст характеристики (обязательно)
  icon?: string; // Класс иконки из иконочного шрифта (опционально)
}
```

## 💅 Кастомизация стилей

#### Базовая структура для CSS

Компонент имеет следующую структуру, которую можно использовать для стилизации:

```html
<ul class="your-class-name">
  <li class="feature-item" data-has-icon="true">
    <span class=" _icon-check" aria-hidden="true"></span>
    <span class="feature-item__text">Текст характеристики</span>
  </li>
  <li class="feature-item" data-has-icon="false">
    <span class="feature-item__text">Текст без иконки</span>
  </li>
</ul>
```

#### CSS селекторы для кастомизации

```css
/* Сам список */
.your-class-name {
  /* ваши стили */
}

/* Каждый пункт списка */
.your-class-name .feature-item {
  /* ваши стили */
}

/* Пункт с иконкой */
.your-class-name .feature-item[data-has-icon='true'] {
  /* ваши стили */
}

/* Пункт без иконки */
.your-class-name .feature-item[data-has-icon='false'] {
  /* ваши стили */
}

/* Иконка */
.your-class-name .feature-item [class*='_icon'] {
  /* ваши стили */
}

/* Текст характеристики */
.your-class-name .feature-item__text {
  /* ваши стили */
}
```

#### Пример кастомизации

```scss
// В вашем компоненте или глобальном файле стилей
.product-features {
  display: flex;
  flex-direction: column;
  gap: 12px;

  .feature-item {
    font-size: 14px;
    color: #333;

    &[data-has-icon='true'] {
      color: #2ecc71; // зеленый цвет для пунктов с иконками
    }

    [class*='_icon'] {
      font-size: 18px;
      color: #3498db; // синий цвет для иконок
    }

    .feature-item__text {
      font-weight: 500;
    }
  }
}
```

## ⚠️ Важные замечания

- Иконки не обязательны - можно передавать только текст
- Пустой массив - если передать пустой массив, компонент ничего не отобразит
- Доступность - иконки автоматически скрываются от скринридеров (они декоративные)
- Иконочный шрифт - классы иконок должны соответствовать вашему иконочному шрифту (например, из icomoon)

## 🔍 Проверка работы

**Если компонент работает правильно, вы увидите:**

- Маркированный список
- Если переданы иконки - они отображаются слева от текста
- Текст переносится на новую строку, если не помещается
- Иконка всегда остается слева от текста

## 🚀 Советы

- Для новичков: начните с простого списка без иконок, потом добавляйте иконки
- Для продакшена: всегда указывайте ariaLabel для лучшей доступности
- Для переиспользования: создавайте отдельные массивы с данными в ваших компонентах
